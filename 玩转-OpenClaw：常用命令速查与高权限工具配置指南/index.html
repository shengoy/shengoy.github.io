玩转 OpenClaw：常用命令速查与高权限工具配置指南

最近大火的开源 AI 助理项目 OpenClaw（前身 Clawd/Moltbot）在 GitHub 上的狂飙突进，让很多人开始将它部署在自己的 VPS 或本地机器上。不过，在最近的 2026.3.x 版本更新后，官方出于安全考虑收紧了默认权限。很多朋友发现升级后，AI 突然”变笨”了，无法执行文件操作或调用本地终端命令。

今天就来帮大家总结一下 OpenClaw 的常用命令，并重点解决如何手动配置 tools.profile 恢复高权限的问题。

一、 核心重点：为什么现在需要手动配置工具权限？

在早期版本中，OpenClaw 默认拥有对宿主机的最高操作权限（比如读写文件、执行 exec 终端命令等）。但作为一个能够直接接入 Telegram、WhatsApp、Discord 等公开通讯平台的 AI Agent，这存在极大的安全隐患。

因此，新版默认关闭了这些高风险工具的直接调用权限。如果你希望你的 AI 助理继续帮你写代码、跑脚本、管理系统，就需要显式地将其工具配置文件（profile）设置为 full 模式。

二、 OpenClaw 常用命令速查表

作为日常维护和折腾，以下这些基础命令是你一定会经常用到的，建议收藏：

  • 系统与初始化

    • openclaw onboard --install-daemon：初始化 API Key 配置并安装后台守护进程（Daemon）。
    • openclaw --version：查看当前安装的 OpenClaw 版本。

  • 网关与运行

    • openclaw gateway --port 18789 --verbose：启动网关并指定端口。加上 --verbose 可以开启详细日志记录，这在排查工具调用失败时非常有用。

  • 配置管理

    • openclaw config path：快速定位当前配置文件的具体路径。
    • openclaw config get <key>：读取某项具体配置（例如排查浏览器配置：openclaw config get browser.defaultProfile）。

  • 诊断与修复

    • openclaw doctor：运行全面的健康检查，诊断内存溢出、权限不足、配置项缺失等潜在问题。
    • openclaw doctor --fix：尝试一键自动修复发现的常见环境问题。
